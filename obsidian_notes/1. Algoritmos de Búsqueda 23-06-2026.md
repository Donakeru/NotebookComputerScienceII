Proceso algorítmico (paso a paso) que localiza un registro especifico de datos en una colección o estructura de datos (llamado x). Son los algoritmos de uso más común de la computación convencional

Algunas estructuras de datos son:
- Listas
- Árboles
- Grafos
### Anatomía de los datos
- **Registro (Record):**  Datos agrupados (edad + nombre + otros)
- **Llave (Key):** campos que pueden identificar de manera única de forma que un algoritmo puede mejorar su desempeño (ej. Cédula, Código, ID) **sería la X mencionada anteriormente**
- La **llave** se vuelve un atributo en la búsqueda

### Búsqueda interna vs Búsqueda externa

**Búsqueda Interna**: Cuando la carga de datos y ejecución de algoritmos se da enteramente en la memoria RAM
-  Es de acceso rápido.
-  Está limitado por ser físicamente volátil y de corta capacidad (5GB usuario promedio)
-  Bajo costo operativo

**Búsqueda Externa**: Ejecutado a través de Almacenamiento secundario (Discos/Archivos)
- Es escalable de manera masiva y no volátil
- Limitado por la velocidad de **transferencia** Input / Output **(I/O)**
- La optimización de algoritmos se encuentra enfocada en la reducción de bloqueo por lecturas de memoria
- Alto costo de transferencia entre almacenamiento secundario a la memoria RAM

### Métricas de desempeño y complejidad

Algunos criterios de evaluación algorítmica pueden ser:

 - Número de comparaciones: Forma principal de evaluación
 - Tiempo de ejecución: Dependencia del entorno físico de ejecución (subjetivo)
 - Operaciones de acceso a disco: Crucial para análisis de bloqueo I/O

### Algunos algoritmos de búsqueda

#### Búsqueda Lineal:
Busca un elemento en un arreglo de objetos.

```c
#include <stdio.h> 

int linear_search(int data[], int size, int target) {

    for (int i = 0; i < size; i++) {
        if (data[i] == target) {
            return i;
        }
    }
    return -1;
}
```

#### Búsqueda Binara:

```c
int binarySearch(int data[], int size, int target) {
    int left = 0;
    int right = size - 1;

    while (left <= right) {
    
        int mid = left + (right - left) / 2; // Si es decimal usar parte entera

        if (data[mid] == target) {
            return mid; // Elemento encontrado
        } else if (data[mid] < target) {
            left = mid + 1; // Buscar en la mitad derecha
        }
        else {
            right = mid - 1; // Buscar en la mitad izquierda
        }
    }
    return -1; // Elemento no encontrado
}
```

#### Búsqueda por Transformación de Llaves (Hashing):

A diferencia de otros algoritmos como los dos anteriores, el hashing rompe un la estructura de datos lineal que tienen los dos anteriores (listas) sino que busca la interpretación de esta como un grafo.

1. **Fundamentos de Hashing:** 
	Es una forma de representación de datos que convierte una llave de búsqueda arbitraria en un entero válido que permita hacer operaciones con complejidad cercana a O(1).
	**Direccionamiento directo vs Tablas Hash:**
	- **Direccionamiento directo:** Caso ideal donde cada posible llave ocuparía únicamente un espacio de memoria. Requiere un espacio proporcional al conjunto universal de llaves lo cual en altos volúmenes de datos ocuparía demasiado espacio 
	- **Tablas Hash:** Piensan en dominios de llaves "una lista con listas" de forma que esté bien distribuidos los datos.
		![[Pasted image 20260623133631.png]] 
		El **factor de carga** es el concepto más importante del hashing. Que es una operación que determina la métrica de densidad operacional de la capacidad de una tabla o cuantos elementos debería tener cada tabla, esto nos ayuda a balancear la carga en las tablas.
		Para un ejemplo de 20 estudiantes (claves a guardar) en una tabla de tamaño 4 nos dice que:
		
		alpha = claves/tamaño; alpha = 20/4 = 5
		
		se debería almacenar 5 estudiantes por key field entre más cercano a 1 el valor de alpha se acerca a lo que sería un direccionamiento directo

1. **Funciones Clásicas de Hash:**
	1. **Método Módulo (Division):** Usa el residuo algebraico de una división entera
		h(key) = key (mod m)
		**Regla de Diseño:** El tamaño de la tabla hash (m) debe ser un número primo y no debe estar cerca de una potencia de 2 para distribuir los índices de manera uniforme.
		**Ejemplo:** a mi código 20212020109 le voy a sacar mod 5 = 4, se almacena el dato con respecto a la tabla de tamaño 5. Organizando la tabla en clase queda muy desbalanceado. A veces incrementar el tamaño de m asegura una mejor dispersión.
	2. **Método mitad de cuadrado:** Elevar al cuadrado las llaves y extraer los dígitos centrales	por ejemplo key = 45^2 = 2025 => extraer mitades => 02 
	3. **Método de Truncado:** Cortar una parte de la llave de forma arbitraria por ejemplo: 20212020109 quedarse con el 1 de ultimo digito de año y 9 de ingreso.
	4. **Conversión de base:** Interpretar una llave original que tiene una base p para convertirlo en una base q y aislar índices estables
2. **Colisiones:** Ocurre cuando dos llaves distintas generan un mismo hash. Existen algunas estrategias que permiten arreglar este problema

3. **Análisis de desempeño de hashing:**
	1. **Caso Promedio O(1):** Asumiendo que la distribución fue uniforme
	2. **Peor de los Casos O(n):** Tendria una revisión de detalles de balanceo y funcion de hashing
